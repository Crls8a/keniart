# Auditoría SEO de Keniart — 2026-07-09

Keniart tiene una base editorial sólida y un frontend rápido, pero todavía no está listo para publicación SEO: las rutas heredan la canonical de la home, faltan `robots.txt` y `sitemap.xml`, el schema mezcla pintura original con reproducciones y existen fallas graves de reflow e imágenes en mobile.

## Alcance y evidencia

- Repositorio: `feature/parallel-frontend-debt-work@84ca6c4`.
- Stack: Next.js 16.2.7, React 19.2.4, TypeScript y Tailwind CSS.
- Sin dominio, deployment, Search Console ni plataforma analítica configurados en el checkout.
- Validación local sobre build productiva: 25 páginas, 34 destinos internos, sin links rotos.
- Lighthouse 13.4 / Chrome 150 en 390×844 y 1280×720.
- Evidencia de performance: `C:\Users\SPARTAN_PC\.codex\audit-results\keniart-2026-07-09`.

## No implementar sin datos confirmados

- No inventar dominio productivo, formación, premios, prensa, exposiciones, precios o plazos.
- No crear una `Organization` si Keniart representa a la artista y no a una empresa independiente.
- No copiar entidades, rutas, analytics IDs ni schema del sitio clínico usado como referencia.
- Search Console, dominio HTTPS, Core Web Vitals de campo y conversiones reales requieren deployment/acceso externo.

## Orden de aplicación

1. Canonicals, metadata, robots, sitemap y schema.
2. Reflow mobile y contrato de imágenes/LCP.
3. Headings, menú y reduced motion.
4. Validación de API y base de medición opt-in.
5. Lint, build, crawl SEO, Lighthouse y revisión mobile/desktop.
