# Runtime data operations

This project has no Dockerfile or Compose stack. Use the local Next.js workflow and keep generated build/cache files out of commits.

## Quick start

```bash
pnpm install
pnpm dev
```

Open `http://localhost:3800`.

## Required environment

No environment example file was found. The current app runs from repository code, `src/data`, and public assets. Add an example env file before introducing required secrets or deployment-specific values.

## Services

| Service | Command | Notes |
|---|---|---|
| Next.js dev server | `pnpm dev` | Runs on port `3800`. |
| Production build | `pnpm build` | Produces `.next/`. |
| Production server | `pnpm start` | Serves the built app. |

## Volumes and data policy

| Path | Policy |
|---|---|
| `src/data/` | Source-controlled sample/domain data; required for the app. |
| `public/` | Source-controlled static assets; required for the app. |
| `public/dossier/dossier-galerias.pdf` | Generated by `pnpm generate:dossier-pdf`; commit only when intentionally updating the deliverable. |
| `.next/`, `node_modules/`, `*.tsbuildinfo` | Disposable local build/cache artifacts. |

## What not to commit

- Local `.env*` files unless a deliberate example file is added.
- `.next/`, `node_modules/`, TypeScript build info, and package-manager debug logs.
- Temporary image optimization output unless it is part of the reviewed asset change.

## Troubleshooting

| Symptom | Check |
|---|---|
| Port already in use | Stop the process using port `3800` or change the script intentionally. |
| Dossier generation fails | Start `pnpm dev` first or pass `DOSSIER_URL` to the generator. |
| Remote images fail | Check `next.config.ts` image host allowlist. |

## Handoff checklist

- [ ] `pnpm install` completed with the committed lockfile.
- [ ] `pnpm dev` opens locally on port `3800`.
- [ ] Generated build/cache files are absent from the diff.
- [ ] Any generated dossier or optimized image asset is intentionally reviewed before commit.
